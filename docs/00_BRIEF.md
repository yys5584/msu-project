# 00. Brief

## 역할

이 문서는 프로젝트의 상위 목적과 제약만 정의한다.

세부 후보는 `docs/03_IDEATION_BACKLOG.md`, 선택된 프로토타입 사양은 `docs/04_PROTOTYPE_DIRECTION.md`, 리서치 지시는 `docs/05_RESEARCH_AGENT_BRIEF.md`에서 관리한다.

## 목적

제한된 기간과 인력 안에서 만들 수 있는 작은 모바일 게임의 핵심 루프를 빠르게 검증하고, 이후 대상 플랫폼에 맞춰 출시 가능한 형태로 확장한다.

초기 목표는 완성형 대작이 아니라 **보이는 핵심 장면**이다.

## 제작 제약

- 1인 개발자 기준
- 3주 MVP 기준
- 웹/앱 기반 개발
- 모바일 스토어 출시 가능성 고려
- 퍼블리싱 지원이 있다고 보고 개발자는 게임 제작에 집중
- 초기 MVP에서는 UI 완성도보다 핵심 루프 검증을 우선

## 제품 가설

유저는 복잡한 시스템보다, 짧은 시간 안에 얻는 시각적 결과와 공유하고 싶은 순간에 먼저 반응한다.

프로토타입은 아래를 증명해야 한다.

- 첫 조작이 단순하다.
- 짧게 플레이해도 결과가 나온다.
- 변화와 보상이 화면에서 보인다.
- 다시 시도할 이유가 있다.
- 모바일 출시 후보로 옮길 수 있다.

## 공개 문서 원칙

공개 문서에는 구체적인 내부 아이디어명, 세부 장르 조합, 민감한 비교 대상을 과도하게 노출하지 않는다.

공개 가능한 수준:

- 문제 정의
- 평가 기준
- 후보 기록 양식
- 프로토타입 구조 템플릿
- 리서치 트랙
- TODO와 미확정 제약

비공개로 관리할 것:

- 구체 후보명
- 독자적인 장르 조합
- 아직 검증되지 않은 내부 아이디어
- 민감한 시장 비교 메모

## 만들지 않을 것

- 대규모 콘텐츠 구조
- 긴 튜토리얼
- 복잡한 계정/경제/상점 시스템
- 3주 안에 구현하기 어려운 멀티플레이/소셜 시스템
- UI 완성도에 의존하는 게임성
