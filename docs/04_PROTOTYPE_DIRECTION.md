# 04. Prototype Direction

> 2026-06-07 기준 첫 프로토타입 방향. 공개 문서에서는 구체 아이디어를 최소화한다.

## 문제 정의

현재 필요한 것은 완성형 장르 선택이 아니라, 30초 안에 플레이-긴장-보상-공유 장면이 보이는 첫 핵심 루프를 고르는 것이다.

## 추천 접근

첫 프로토타입은 아래 조건을 만족해야 한다.

- 단일 화면 또는 매우 적은 화면 수
- 첫 조작 1개
- 10~30초 안에 결과 도달
- 결과 화면에서 보상, 변화, 희귀성이 즉시 보임
- 낮은 UI 완성도에서도 재미 판단 가능
- 3주 안에 구현, 테스트, 출시 빌드 준비 가능

## 첫 프로토타입 루프 템플릿

1. 시작
2. 짧은 진행
3. 한 번의 선택 또는 자동 판정
4. 결과 변화
5. 보상/결과 공개
6. 재도전

## 화면 기준

첫 화면:

- 유저가 눌러야 할 행동이 하나만 보인다.
- 다음에 무슨 일이 일어날지 시각적으로 예상할 수 있다.

진행 화면:

- 상태 변화가 즉시 보인다.
- 텍스트 설명보다 모션과 숫자 변화를 우선한다.

결과 화면:

- 결과 하나를 가장 크게 보여준다.
- 성공/보통/실패 또는 희귀/일반이 즉시 구분된다.
- 공유용 캡처에 필요한 정보만 남긴다.

## 최소 데이터 구조

- `RunConfig`: 진행 시간, 난이도, 보상 가중치
- `ResultItem`: 이름, 등급, 시각 키, 설명
- `RunResult`: 선택/판정, 점수, 획득 결과, 반복 보너스

## 검증 기준

- 첫 플레이가 30초 안에 끝나는가?
- 15초 영상으로 시작-변화-결과가 전달되는가?
- 낮은 결과도 보기 좋은가?
- 높은 결과는 한눈에 다른가?
- 결과 화면 한 장으로 특별함이 보이는가?

## 당장 만들지 않을 것

- 계정/인벤토리
- 장기 강화
- 상점
- 커뮤니티 보드
- 실제 플랫폼 연동
- 복잡한 전투 규칙
