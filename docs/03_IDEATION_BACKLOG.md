# 03. Ideation Backlog

> 아이디어는 확정안이 아니라 검증 후보로 관리한다.

## 공개 문서 원칙

구체적인 내부 아이디어명과 장르 조합은 공개 문서에 자세히 쓰지 않는다.

이 문서는 후보를 평가하는 기준만 남기고, 실제 후보 목록은 비공개 리서치 또는 작업 문서에서 관리한다.

## 평가 기준

각 아이디어는 1~5점으로 평가한다.

- 즉시 이해: 설명 없이 이해되는가?
- 보이는 재미: 화면/영상으로 재미가 보이는가?
- 자랑 가능: 결과를 공유하고 싶은가?
- 신규 유저 친화: 첫날 유저도 의미 있는 결과가 가능한가?
- 구현 속도: 빠르게 만들 수 있는가?
- 출시 적합성: 모바일 출시 준비가 현실적인가?

## 후보 기록 형식

```md
## 후보명

### 핵심 장면

### 첫 조작

### 30초 결과

### 반복 이유

### 공유 가능한 화면

### 구현 리스크

### 제외할 범위
```

## 현재 상태

구체 후보는 리서치 이후 다시 정리한다.

현재 우선순위는 후보 확정이 아니라, 3주 1인 MVP 기준에서 가능한 플레이 루프의 조건을 좁히는 것이다.
