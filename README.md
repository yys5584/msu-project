# MSU Project

짧은 모바일 게임 MVP를 기획하고 검증하기 위한 문서 기반 작업 폴더다.

## Purpose

이 저장소의 목적은 제한된 기간과 인력 안에서 만들 수 있는 작은 게임의 핵심 루프를 정리하고, 리서치와 프로토타입 방향을 문서화하는 것이다.

초기 목표는 완성형 대작이 아니라 다음을 빠르게 확인하는 것이다.

- 첫 조작이 단순한가
- 짧은 시간 안에 결과가 나오는가
- 화면만 봐도 보상과 변화가 보이는가
- 반복 플레이할 이유가 있는가
- 모바일 출시 후보로 옮길 수 있는가

## Project Constraints

- 개발 인원: 1인 개발자 기준
- 기간: 3주 MVP 기준
- 작업 방식: 문서, 리서치, 프로토타입을 병렬로 정리
- 타겟: 웹/앱 기반 모바일 게임
- 출시 방향: 모바일 스토어 배포 가능성을 고려
- 우선순위: UI 완성도보다 핵심 루프 검증

## Decision Criteria

아이디어는 아래 기준으로 검토한다.

1. 3주 안에 핵심 루프를 구현할 수 있는가
2. 첫 화면에서 해야 할 일이 보이는가
3. 첫 결과가 빠르게 나오는가
4. 반복할수록 변화가 보이는가
5. 모바일 조작에 맞는가
6. 플랫폼 이관과 출시 준비가 현실적인가
7. 기존 시장 사례와 비교했을 때 차별점이 있는가

## Research Direction

리서치는 공개 자료와 커뮤니티 반응을 바탕으로 진행한다.

중점:

- 최근 모바일/캐주얼 게임 흐름
- 짧은 플레이 루프가 강한 사례
- 커뮤니티에서 반응이 좋았던 장면
- 작은 팀이 만들 수 있는 범위
- 출시 단계에서 문제가 될 수 있는 범위

## Output Rules

- 중요한 판단은 문서에 남긴다.
- 미확정 내용은 단정하지 않고 TODO로 남긴다.
- 프로토타입은 작게 만든다.
- 공개 저장소에는 구체적인 내부 아이디어를 과도하게 노출하지 않는다.
